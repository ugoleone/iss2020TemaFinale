Introduction

In this work, we will develop a Web-based user-interface for the basicrobot introduced in
Qak20Basicrobot | The basicrobot using QAkactors.

More precisely, we will operate as follows:

  1. Experiment on a PC

    1. Activate our virtual robot on a PC (address 192.168.1.pc).
    2. Build (project it.unibo.qak20.basicrobot) the distribution file for our basicrobot.
    3. Deploy the distribution on the PC and activate the basicrobot.
    4. Send some command to the basicrobot by using a conventional GUI written in Java (e.g. consoleGuiCoap.kt)
    5. Working in this project (robotWeb2020), set the proper values in the configuration file configurator.kt.
      Now, activate the Spring application RobotWeb2020Application.java on the PC, open a browser on localhost:8080 (or 192.168.1.pc:8080) and send some command to the basicrobot.


    6. Activate a CoAP client working as an observer of the basicrobot resource, e.g. actorQakCoapObserver.kt, move the basicrobot and notice that it shows, after a command, the same values that appears in the Resource state section of the web-page.
    7. Optionally, change the HTML page to be displayed, by changing the String htmlPage in RobotController.java. For example, the previous page (robotGuiSocket) sends commands by using a socket.io, while the following one (robotGuiPost) does use the conventional POST HTTP-verb :

  2. Run the basicrobot on the RaspberryPi

    1. Deploy the distribution file for the basicrobot on a RaspberryPi (address 192.168.1.rasp).
    2. Set the proper values in the basicrobotConfig.json: 
      {"type":"virtual", "port":"8999", "ip":"192.168.1.pc"}
    3. Activate the basicrobot (bash it.unibo.qak20.basicrobot)
    4. In project robotWeb2020, change the configuration values in the file configurator.kt 
      public static final String hostAddr   	    = "192.168.1.rasp";   	
public static final String port    			= "8020";
public static final String qakdestination	= "basicrobot";
public static final String ctxqadest 		= "ctxbasicrobot";
    5. Move the robot by using the Web-interface

  3. Use a real robot



    		4.

    1. On the RaspberryPi, change the configuration values in the file basicrobotConfig.json: 
      {"type":"realmbot", "port":"/dev/ttyUSB0", "ip":"dontcare"}
    2. Activate the basicrobot as before (bash it.unibo.qak20.basicrobot)
    3. Move the robot by using the Web-interface configured as before
